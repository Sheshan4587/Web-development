Accessibility

Ensuring web accessibility is crucial to making the Content Page usable for all users, including those with disabilities. This page incorporates several HTML and CSS techniques to enhance readability, navigation, and overall user experience. One of the primary accessibility features implemented is the use of semantic HTML elements. The page structure follows a logical hierarchy, with elements like header, nav, section, footer, and h1 to h4 used correctly. This ensures that screen readers and assistive technologies can easily interpret and navigate the content. The nav element contains an unordered list ul with properly structured a links, allowing keyboard users to move through the site efficiently using the Tab key. Images play a crucial role in conveying information, and accessibility is improved by including descriptive alt attributes for all images. For example, "farmer cultivating", could be used to provide context to visually impaired users. This ensures that those who rely on screen readers still understand the visual content. Additionally, ensuring that the images scale responsively (width: 100%; height: auto;) prevents overlapping or layout-breaking issues on different devices.

Challenges and Lessons Learned

Another challenge was implementing form validation and ensuring accessibility for the feedback form. Initially, the form lacked validation, and it wasn’t fully accessible to screen readers or keyboard users. I added HTML5 validation attributes like required and pattern for phone numbers, and improved accessibility by linking labels to inputs and ensuring all elements were focusable with the Tab key. These changes made the form more functional and inclusive.

Finally, maintaining consistent styling across pages was tricky. The team and content pages had different spacing, making the site feel disjointed. To fix this, I created a shared CSS file (style.css) for global styles and used reusable classes for buttons, cards, and headings. This ensured a cohesive design across the website.

Compliance

The Feedback, Team, and Content pages have been developed with careful consideration of the Janet Web Regulations, which govern web page publication in the UK. These regulations emphasize accessibility, usability, and security, and I have made efforts to ensure compliance with these standards. Below, I discuss how the pages align with these regulations.

Accessibility

Janet regulations require websites to be accessible to all users, including those with disabilities. To meet this requirement, I used semantic HTML elements like header, nav, and footer to structure the content logically, making it easier for screen readers to interpret. All interactive elements, such as form fields and buttons, are focusable and navigable using the Tab key, ensuring usability for keyboard-only users. Additionally, all images include descriptive alt attributes, providing context for visually impaired users. Text colors were chosen to ensure sufficient contrast against backgrounds, and font sizes are adjustable for better readability.

Usability

Janet regulations emphasize creating user-friendly websites. To comply, I ensured the feedback form includes HTML5 validation (e.g., required, type="email", and pattern for phone numbers) to prevent users from submitting incomplete or incorrect data. Clear labels and instructions were added to guide users through the form, improving overall usability.

Compliance with Standards

The pages adhere to HTML5 standards, ensuring compatibility with modern browsers. I also tested the pages using tools like W3C Validator to ensure the code is free of errors and follows best practices. While the pages are not formally audited for Janet compliance, they reflect a strong effort to align with the regulations.
